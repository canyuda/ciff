# 熔断器使用方式

## 概述

每个 LLM Provider 拥有独立的熔断器实例，通过 `CircuitBreakerService` 统一管理。调用时自动获得熔断保护 + 重试能力。

## 熔断器配置

| 参数 | 默认值 | 说明 |
|------|-------|------|
| slidingWindowSize | 10 | 滑动窗口大小（最近 10 次调用） |
| failureRateThreshold | 50% | 失败率达到 50% 时熔断开启 |
| waitDurationInOpenState | 30s | 熔断开启后等待 30s 进入半开 |
| permittedNumberOfCallsInHalfOpenState | 3 | 半开状态允许 3 次探测调用 |

配置位于 `application.yml`：

```yaml
ciff:
  resilience:
    circuit-breaker:
      sliding-window-size: 10
      failure-rate-threshold: 50
      wait-duration-in-open-state: 30s
      permitted-number-of-calls-in-half-open-state: 3
```

## 熔断状态流转

```
CLOSED（正常）
  │  失败率 ≥ 50%（10 次窗口内）
  ▼
OPEN（熔断，拒绝所有请求）
  │  等待 30s
  ▼
HALF_OPEN（半开，允许 3 次探测）
  │  探测成功 → 回到 CLOSED
  │  探测失败 → 回到 OPEN
  ▼
```

## 失败记录规则

通过 `recordException` 谓词过滤：

| 异常类型 | 计入失败率 | 原因 |
|---------|-----------|------|
| TIMEOUT | 是 | 瞬态错误，可能持续 |
| RATE_LIMITED | 是 | 瞬态错误，可能持续 |
| UNKNOWN | 是 | 服务端异常 |
| AUTH_FAILED | **否** | 配置错误，不是瞬态问题 |

## 重试策略

| 异常类型 | 最大重试次数 | 间隔策略 |
|---------|------------|---------|
| TIMEOUT | 2 次（共 3 次尝试） | 固定 1s |
| RATE_LIMITED | 2 次（共 3 次尝试） | 指数退避 2s → 4s |
| AUTH_FAILED | 0 次 | 不重试 |
| UNKNOWN | 0 次 | 不重试 |

## 使用示例

### 注入依赖

```java
@Service
@RequiredArgsConstructor
public class LlmService {

    private final LlmHttpClient llmHttpClient;
    private final CircuitBreakerService circuitBreakerService;
}
```

### 同步调用

```java
public String chat(String providerName, String url,
                   Map<String, String> headers, String requestBody) {

    return circuitBreakerService.execute(providerName, () ->
            llmHttpClient.post(url, headers, requestBody));
}
```

### 流式调用（SSE）

```java
public void chatStream(String providerName, String url,
                       Map<String, String> headers, String requestBody,
                       SseEmitter emitter) {

    circuitBreakerService.execute(providerName, () -> {
        llmHttpClient.stream(url, headers, requestBody, data -> {
            try {
                emitter.send(SseEmitter.event().data(data));
            } catch (IOException e) {
                throw new LlmApiException(LlmApiException.ErrorType.UNKNOWN,
                        "SSE send failed", e);
            }
        });
        return null;
    });
}
```

### 获取熔断器状态

```java
CircuitBreaker cb = circuitBreakerService.getOrCreate("openai");

cb.getState();              // CLOSED / OPEN / HALF_OPEN
cb.getMetrics().getFailureRate();       // 当前失败率
cb.getMetrics().getNumberOfFailedCalls(); // 失败调用数
cb.getMetrics().getNumberOfSuccessfulCalls(); // 成功调用数
```

## 调用链路

```
Controller
  └── circuitBreakerService.execute("openai", action)
       └── CircuitBreaker (CLOSED?)
            ├── OPEN → throw CallNotPermittedException（快速失败）
            └── CLOSED / HALF_OPEN → executeWithRetry(action)
                 ├── attempt 1 → action.get()
                 │    ├── success → CB record success → return result
                 │    └── LlmApiException(TIMEOUT) → sleep 1s → attempt 2
                 │         ├── success → CB record success → return result
                 │         └── LlmApiException(TIMEOUT) → sleep 1s → attempt 3
                 │              ├── success → CB record success → return result
                 │              └── fail → CB record failure → throw
                 └── LlmApiException(AUTH_FAILED) → no retry → CB ignore → throw
```
